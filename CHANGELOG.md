## 版本说明

### 语义化版本控制

- **主版本号**: 不兼容的 API 修改
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 变更类型

- `🎉` 新功能 (Added)
- `🔧` 改进 (Changed)
- `🐛` Bug 修复 (Fixed)
- `⚡` 性能优化 (Performance)
- `🏗️` 重构 (Refactoring)
- `✨` 文档更新 (Documentation)
- `💥` 破坏性变更 (Breaking Change)

### 技术栈演进

- **Three.js**: 从早期版本升级到 v0.171
- **TypeScript**: 全面采用，严格模式
- **构建工具**: 从 Rollup 迁移到 Vite
- **包管理**: 单包模式改为 npm workspaces
- **发布渠道**: GitHub + npm 双轨发布

## 贡献指南

### 提交规范

请遵循 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/v1.0.0/) 规范：

```bash
feat: 新功能
fix: Bug 修复
docs: 文档更新
style: 代码格式修改
refactor: 代码重构
perf: 性能优化
test: 测试相关
chore: 构建过程或辅助工具的变动
```

### 版本发布流程

1. 合并所有待发布的代码
2. 更新版本号
3. 更新 CHANGELOG.md
4. 创建 Git 标签
5. 发布到 npm
6. 创建 GitHub Release
